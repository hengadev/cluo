# Lovable

I need you to design visually consistent, professional document templates for a web or PDF-based system.
Each document type should have its own layout and structure but share a unified visual identity (fonts, margins, spacing, header/footer style, etc.).

Please create mockups or layout designs for the following five document types:

Invoice (Facture) – Used for billing clients. Include company logo area, invoice number/date, sender and recipient details, itemized list with quantity, description, unit price, total, and payment instructions.

Mandate (Mandat) – Used to authorize a service provider to act on behalf of a client. Include signature areas for both parties, identification details, purpose of the mandate, validity period, and legal notes.

Estimate (Devis) – A quotation before performing a service. Include company details, estimate number/date, client info, description of services or products, quantities, unit price, total, and validity period.

Contract (Contrat) – A formal agreement between two parties. Include title, introduction, sections for clauses (e.g., “Purpose”, “Duration”, “Payment Terms”, “Confidentiality”, “Termination”), and signature sections for both parties.

Design requirements:

Minimal, clean, and modern layout (think legal or administrative document style, not flashy marketing design)

Suitable for both PDF export and on-screen viewing

Use a neutral color palette (black, white, greys, subtle accent color)

Maintain the same header/footer styling across all document types

Make sure each design supports dynamic data (company info, user info, items, etc.)

Output expectations:

One layout preview per document type

Optional: typography and color palette recommendations

Optional: consistent reusable components (e.g., “header”, “signature section”, “item table”)

The goal is to have ready-to-implement templates that can later be integrated into a SvelteKit or Golang PDF rendering workflow.
